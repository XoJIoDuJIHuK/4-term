Сетевая архитектура "клиент-сервер" - это распределенная сетевая архитектура, в которой клиент и сервер взаимодействуют друг с другом для выполнения определенных задач.

Клиентская часть приложения предназначена для взаимодействия с пользователем и отправки запросов на сервер, а серверная часть обрабатывает запросы от клиента и возвращает результаты обратно.

Сетевой протокол - это набор правил и форматов, которые определяют способ передачи данных между устройствами в сети.

Стек протоколов - это набор протоколов, которые используются для обработки и передачи данных в сети.

В стек протоколов TCP/IP входят следующие протоколы: IP, TCP, UDP, ICMP, ARP, DNS.

Сокет - это программный интерфейс для обмена данными между устройствами в сети, который используется для установления соединения между клиентом и сервером.

Номер порта - это числовой идентификатор, который используется для указания конкретного сетевого приложения, с которым необходимо установить соединение.

Для установления сокетного соединения необходимо выполнить следующие шаги: создать сокет клиента, указать адрес и номер порта сервера, установить соединение с сервером, отправить и получить данные.

Датаграмма - это небольшой пакет данных, который передается по сети без установления соединения и гарантии доставки.

RMI (Remote Method Invocation) - это технология, которая позволяет вызывать методы удаленных объектов на другом компьютере через сеть.
